永远不要随意修改 edge 浏览器里的 “使用安全的 DNS” 选项。（扶额）

从香港回内地之后，发现有自己的电脑有特定的网址打不开，但是其他设备是可以打开该特定的网址的。于是就在猜可能是电脑又出现了各种网络问题，之前刚来香港的时候也有出现类似的问题，但是当时搞了半天最后做出的修复步骤是重装系统（

我想着要是这一次还是得重装系统就太麻烦了。然后和 GPT 对了半天的轴。发现是 edge 浏览器里的 “使用安全的 DNS” 选项选择了 cloudflare 的特定 DNS 作为 IP 解析，而刚好我在的内地网络阻挡了该类型的 IP。使电脑在尝试连接该网址的时候出现：

> **TCP 三次握手成功，ClientHello 成功发出，但服务器（或者位于服务器前面的设备）在收到 ClientHello 后立即返回 RST。**

的情况。并且由于未知原因，这个修改是会被 `curl` 缓存下来的，结果就是 `nslookup` 的时候显示的是一套 `IP`，但是 `curl` 调用的是另一个 `IP`，然后 `curl` 就也联不通该网址。总之，不要瞎改（

:::note
我甚至在猜说，去年的网络情况可能就是由于不知道什么时候配置了这个 `DNS` 所导致的 = =
如果是这样，那确实没招了（
